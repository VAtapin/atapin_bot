# Онлайн-оплата и подписки

## Stripe

Сервис использует настоящий ежемесячный Stripe Checkout в режиме `subscription`.
Первый платёж и последующие автоматические продления подтверждаются только
подписанными webhook-событиями. Возврат пользователя из Stripe сам по себе не
активирует тариф.

### Настройки платформы

Откройте **Админка → Интеграции → Настройки платформы** и заполните раздел
«Платежи»:

1. `Платёжный провайдер`: `stripe`.
2. `Тестовый режим`: `1` для ключа `sk_test_…`, `0` для `sk_live_…`.
3. `Секретный ключ провайдера`: Stripe Secret key.
4. `Shop ID / Account ID`: для Stripe оставить пустым.
5. `Секрет webhook`: Stripe Signing secret `whsec_…`.
6. После проверки включить `Включить онлайн-платежи`.

Для каждого тарифа желательно заполнить **Stripe Price ID** (`price_…`):
создайте в Stripe Product с ежемесячной recurring Price. Это позволяет менять
тариф действующей подписки с корректным перерасчётом. Первая подписка может быть
создана и без Price ID, но смена такого тарифа потребует заполнить ID нового
тарифа.

Секреты после сохранения не выводятся. Пустое поле при повторном сохранении не
стирает уже записанный секрет.

### Webhook

Endpoint:

```text
https://idommoy.com/api/payments/webhook/stripe
```

В Stripe Dashboard откройте **Developers → Webhooks → Add endpoint** и включите:

- `checkout.session.completed`;
- `customer.subscription.created`;
- `invoice.paid`;
- `invoice.payment_failed`;
- `invoice.payment_action_required`;
- `customer.subscription.updated`;
- `customer.subscription.deleted`;
- `charge.refunded`.

Скопируйте Signing secret созданного endpoint (`whsec_…`) в настройку
«Секрет webhook». Для test и live режима Stripe создаются разные endpoint и
разные signing secret.

### Что происходит

- Checkout создаёт ежемесячную подписку.
- `invoice.paid` активирует тариф и переносит дату следующей оплаты.
- `invoice.payment_failed` переводит подписку в `past_due` и даёт 7 дней
  льготного периода.
- Отмена из панели владельца передаётся в Stripe как
  `cancel_at_period_end`: доступ сохраняется до конца оплаченного периода.
- Удаление подписки и возврат синхронизируются webhook-событиями.
- Все входящие события сохраняются в журнале webhook и обрабатываются
  идемпотентно.

### Проверка

В тестовом режиме используйте тестовую карту Stripe `4242 4242 4242 4242`,
любую будущую дату и любой CVC. После оплаты проверьте:

- подписку дерева;
- запись платежа;
- журнал webhook;
- дату следующего списания.

Не активируйте рабочий режим до успешной тестовой оплаты и получения всех
webhook-событий.

## ЮKassa и ручная оплата

Существующий код ЮKassa поддерживает разовый платёж с подтверждением webhook.
Автоматическое продление реализовано только для Stripe. Значение `manual`
создаёт заявку, которую подтверждает суперадминистратор.
